# Backend Architecture

## Overview

The Viscas Backend is designed using a modular, service-oriented architecture that separates business logic, authentication, AI processing, third-party integrations, and data persistence into independent layers.

The backend acts as the central intelligence hub of the platform. All communication between the frontend and external services such as Gmail, Google Calendar, and Corsair AI is routed through the backend to ensure security, consistency, and maintainability.

The architecture follows an API-First approach, allowing multiple clients (Web, Mobile, Desktop, Browser Extension) to consume the same backend services.

---

# Layer Responsibilities

## API Layer

Responsible for:

- Request Routing
- Request Parsing
- Response Formatting
- Status Codes

---

## Middleware Layer

Responsibilities:

- Authentication
- Authorization
- Validation
- Rate Limiting
- Logging
- Error Handling

---

## Service Layer

Contains all business logic.

Services include:

- Auth Service
- Gmail Service
- Calendar Service
- AI Service
- Task Service
- Subscription Service
- Notification Service (Future)

Business rules should exist only inside this layer.

---

## Integration Layer

Responsible for communication with external providers.

Current Integrations:

- Gmail API
- Google Calendar API
- Corsair AI

Future Integrations:

- Google Drive
- Slack
- Outlook
- Notion
- Zoom

---

## Data Layer

Responsible for:

- Database Queries
- Transactions
- Relationships
- Data Integrity

Implemented using Prisma ORM.
