# Corsair AI Architecture

## Overview

Corsair AI serves as the primary Artificial Intelligence engine powering the intelligent capabilities of Viscas.

Rather than interacting directly with the AI provider from the frontend, all AI requests are processed through the Viscas Backend. This architecture ensures security, context awareness, request validation, and centralized AI orchestration.

The AI layer is designed to be provider-independent, allowing future integration with multiple AI providers such as OpenAI, Google Gemini, and Anthropic Claude.

# Context Builder

The Context Builder gathers only the information required to complete the user's request.

Possible Context Sources:

- Gmail
- Google Calendar
- Tasks
- User Preferences
- Current Screen
- Conversation History
- Subscription Plan

The system follows the **Principle of Minimum Context**, ensuring that unnecessary user data is never sent to the AI provider.

---

# Prompt Builder

The Prompt Builder converts application data into structured prompts.

Responsibilities:

- Normalize data
- Remove unnecessary information
- Define AI instructions
- Attach user context
- Optimize token usage

Example Workflow:

```
Email Content
      │
      ▼
Extract Relevant Information
      │
      ▼
Generate Prompt
      │
      ▼
Corsair AI
```

# Security

The AI architecture follows strict security practices:

- Backend-only AI communication
- API key protection
- Prompt validation
- Response sanitization
- Sensitive data filtering
- HTTPS communication

No AI API keys are exposed to the frontend.

---

# Error Handling

Possible failures include:

- AI Timeout
- Invalid Prompt
- API Rate Limit
- Network Failure
- Empty Response
- Provider Unavailable

Fallback responses should provide meaningful feedback to the user without exposing internal errors.
