# Calendar User Flow

## Overview

This document describes how users interact with the Calendar module and how every request flows through the Viscas platform.

All Calendar operations are processed through the Viscas Backend before communicating with Google Calendar APIs, ensuring centralized authentication, validation, security, and AI integration.

# API Mapping

| User Action | API | Service |
|-------------|-------------------------------|----------------------|
| Load Events | GET /api/v1/calendar/events | Google Calendar API |
| View Event | GET /api/v1/calendar/event/:id | Google Calendar API |
| Create Event | POST /api/v1/calendar/events | Google Calendar API |
| Update Event | PATCH /api/v1/calendar/events/:id | Google Calendar API |
| Delete Event | DELETE /api/v1/calendar/events/:id | Google Calendar API |
